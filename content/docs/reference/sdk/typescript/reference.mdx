Sourced from `@vendodev/sdk`. Every public method on the `Vendo` client is
documented here, plus the subpath modules (`reconciler`) and top-level helpers.

Every request method is `async` and returns a `Promise`. The two exceptions are
`invalidate(slug)` (synchronous cache eviction) and `connectUrl(slug, opts?)`
(URL builder).

```typescript
import { Vendo } from "@vendodev/sdk";
const vendo = new Vendo();
```

---

## client

Top-level methods on the client instance itself.

### isVendoMode

Returns `true` when the SDK is running in Vendo mode (`VENDO_API_KEY` is set).

```typescript
vendo.isVendoMode(): boolean
```

```typescript
if (vendo.isVendoMode()) {
  const balance = await vendo.billing.balance();
}
```

---

### forUser

Returns a new client instance scoped to a user JWT.

```typescript
vendo.forUser(userJwt: string): Vendo
```

**Throws:** `VendoOnlyFeature` — in OSS mode.

```typescript
const scoped = vendo.forUser(userJwt);
const conn = await scoped.connections.get("stripe");
```

---

### forRequest

Returns a new client scoped to the `X-Vendo-User-JWT` extracted from an incoming request's headers.

```typescript
vendo.forRequest(headers: Headers | Record<string, string>): Vendo
```

**Throws:**
- `VendoOnlyFeature` — in OSS mode.
- `IdentityNotPresent` — when `X-Vendo-User-JWT` is absent from the headers.

```typescript
const scoped = vendo.forRequest(request.headers);
const conn = await scoped.connections.get("notion");
```

### token

Returns the credential token for a provider slug. In Vendo mode this fetches a refreshed OAuth token; in OSS mode it reads the conventional env var.

```typescript
vendo.token(slug: string): Promise<string>
```

**Throws:** `NotConnected` — when no token is found.

```typescript
const openaiKey = await vendo.token("openai");
```

---

### tokens

Returns tokens for multiple slugs in one call.

```typescript
vendo.tokens(slugs: string[]): Promise<Record<string, string | null>>
```

```typescript
const tokens = await vendo.tokens(["openai", "anthropic"]);
// { openai: "sk-...", anthropic: null }
```

---

### invalidate

Synchronously evicts the cached token for a slug. The next `token()` call re-fetches.

```typescript
vendo.invalidate(slug: string): void
```

```typescript
vendo.invalidate("google");
```

---

### connectUrl

Returns the OAuth connect URL for a provider slug.

```typescript
vendo.connectUrl(slug: string, options?: { returnTo?: string; state?: string }): string
```

**Throws:** `VendoOnlyFeature` — in OSS mode.

```typescript
const url = vendo.connectUrl("stripe", { returnTo: "https://myapp.vendo.run/dashboard" });
```

## connections

Read the user's provider connections for this deployment.

### connections.list

Returns all connections for this deployment, both connected and available.

```typescript
vendo.connections.list(): Promise<Connection[]>
```

Concurrent calls share one in-flight HTTP request.

```typescript
const all = await vendo.connections.list();
const connected = all.filter(c => c.status === "connected");
```

---

### connections.get

Returns a single connection by provider slug, or `null` if the slug is absent from this deployment's catalog.

```typescript
vendo.connections.get(slug: string): Promise<Connection | null>
```

`null` means the slug is not in this deployment's catalog at all. An
"available" connection (catalog-listed but unbound) is still returned with
`status === "available"` — check the status to distinguish.

```typescript
const conn = await vendo.connections.get("supabase");
if (!conn) return { error: "supabase not in catalog" };
if (conn.status !== "connected") return { needsConnection: "supabase" };
const ref = conn.context?.["ref"];
```

## events

Server-side SSE event stream. Vendo mode only.

### events.subscribe

Opens a persistent SSE stream and yields events as they arrive. Reconnects with exponential backoff (capped at 30 s by default).

```typescript
vendo.events.subscribe(options?: SubscribeOptions): AsyncIterable<EventStreamMessage>
```

```typescript
interface SubscribeOptions {
  signal?: AbortSignal;
  maxBackoffMs?: number;  // default 30_000
  onReconnect?: (attempt: number, err?: Error) => void;
}
```

**Throws:** `VendoOnlyFeature` — in OSS mode.

```typescript
for await (const event of vendo.events.subscribe()) {
  if (event.type === "connection.connected") {
    const slug = (event.data as { slug?: string }).slug;
    if (slug) vendo.invalidate(slug);
  }
}
```

## webhooks

Verify inbound webhook signatures. Works in any mode.

### webhooks.verify

Verifies the HMAC signature on an inbound webhook and returns the parsed event.

```typescript
import { WebhooksAPI } from "@vendodev/sdk";

const hooks = new WebhooksAPI({ secret: "whsec_...", maxAgeSec: 300 });
hooks.verify(headers: Headers | Record<string, string>, body: string): WebhookEvent
```

`secret` and `maxAgeSec` are **constructor** parameters on `WebhooksAPI`.
`verify()` takes only `(headers, body)`. `body` is a `string`.

When the SDK is constructed without an explicit secret, the verifier reads
`VENDO_WEBHOOK_SECRET` from `process.env`.

**Throws:** `ValidationError` — signature missing, timestamp out of window, body not JSON, or HMAC mismatch.

```typescript
const event = vendo.webhooks.verify(request.headers, rawBody);
if (event.type === "connection.connected") { ... }
```

## data

Execute Composio-backed actions against connected providers. Vendo mode only.

### data.execute

Posts an action to the Composio proxy and returns the unwrapped result.

```typescript
vendo.data.execute(action: string, args?: Record<string, unknown>): Promise<unknown>
```

Posts to `https://composio-proxy.vendo.run/v1/execute` (override via
`VENDO_DATA_PROXY_URL`).

**Throws:**
- `NotConnected` (`binding_missing`) — when the provider required by the action is not connected.
- `VendoOnlyFeature` — in OSS mode.
- `UpstreamError` (`composio_error`) — when Composio returns an error.
- `AuthError` — when `VENDO_API_KEY` is missing or invalid.

```typescript
const result = await vendo.data.execute(
  "SUPABASE_BETA_RUN_SQL_QUERY",
  { project_id: "vqxkddreoeebocpisgrn", query: "SELECT count(*) FROM auth.users" }
);
```

## reconciler (subpath module)

`@vendodev/sdk/reconciler` is a Node-only subpath module — **not** a property on
the `Vendo` class. It is a separate import.

```typescript
import * as reconciler from "@vendodev/sdk/reconciler";
```

### reconciler.bootstrap

Writes a `.env` file once from the current connection state.

```typescript
reconciler.bootstrap(options: ReconcilerOptions): Promise<void>
```

```typescript
interface ReconcilerOptions {
  envFile: string;
  mapping: () => Promise<Record<string, string>> | Record<string, string>;
  onChange?: "restart" | "reload-hook" | ((diff: ReconcilerDiff) => void | Promise<void>);
  reloadUrl?: string;
  pollIntervalMs?: number;
}
```

`mapping` is a **callable** returning the env dict — not the dict itself.

```typescript
await reconciler.bootstrap({
  envFile: "/app/.env",
  mapping: async () => ({ OPENAI_API_KEY: await vendo.token("openai") }),
});
```

---

### reconciler.start

Starts a long-running watcher that polls `mapping()` every `pollIntervalMs` (default 30 s) and applies the diff to the env file.

```typescript
reconciler.start(options: ReconcilerOptions): Promise<Reconciler>
```

**Returns:** an object with a `stop(): void` method.

```typescript
const r = await reconciler.start({
  envFile: "/app/.env",
  mapping: async () => ({ OPENAI_API_KEY: await vendo.token("openai") }),
  onChange: "restart",
});
// Later:
r.stop();
```

`onChange` values:
- `"restart"` — `process.exit(0)` (your supervisor restarts the container).
- `"reload-hook"` — POST `{ changed: [...] }` to `reloadUrl`.
- `(diff) => void | Promise<void>` — call your custom handler.

## Types

### Connection

Represents a provider connection for this deployment.

| Field | Type | Description |
|---|---|---|
| `id` | `string` | UUID. Required. |
| `externalId` | `string` | Provider-side identifier (user ID, account ID, etc.). Required. |
| `slug` | `string` | Provider slug. |
| `displayName` | `string` | Human-readable provider name. |
| `category` | `string` | Provider category. |
| `profile` | `string` | Credential management mode. |
| `status` | `"connected" \| "available" \| "pending_setup" \| "needs_reauth" \| "error" \| "revoked"` | Connection state. |
| `metadata` | `Record<string, unknown>` | Raw provider metadata. Prefer `context` for discovery identifiers. |
| `context` | `Record<string, unknown> \| null` | Flattened discovery identifiers (e.g. `{ ref: "vqxkddreoeebocpisgrn" }` for Supabase). `null` for non-discovery providers. |
| `credential` | `Record<string, string \| number> \| null` | Per-provider credential payload (used by `vendo_managed_pool` connections). |
| `setupUrl` | `string \| null` | OAuth connect URL. Set on `available` entries. |
| `errorMessage` | `string \| null` | Human-readable error when `status` is `"error"` or `"needs_reauth"`. |
| `connectedAt` | `string \| null` | ISO 8601 timestamp. |
| `logoUrl` | `string` | Provider logo URL (empty string when missing). |
| `brandColor` | `string` | Provider brand color hex (empty string when missing). |
| `docsUrl` | `string` | Provider docs URL (empty string when missing). |
| `envBootstrap` | `EnvBootstrap \| null` | Env-var bootstrap spec for the reconciler. |

**Context field by provider:**

| Provider | `context` shape |
|---|---|
| `supabase` | `{ ref: "<project-ref>" }` |
| `instagram` | `{ ig_user_id: "<id>" }` |
| Slack, Gmail, Stripe, etc. | `null` |

## Errors

| Class | Extends | When thrown |
|---|---|---|
| `VendoError` | `Error` | Base class for all SDK errors. |
| `AuthError` | `VendoError` | `app_unknown` / `app_revoked` / `app_expired`. |
| `NotConnected` | `VendoError` | `binding_missing` / `connection_revoked`. Carries `slug`, `suggestedFix`. |
| `NeedsReauth` | `VendoError` | `connection_needs_reauth`. Carries `connectUrl`. |
| `BalanceExhausted` | `VendoError` | `balance_exhausted`. |
| `SpendCapExceeded` | `VendoError` | `spend_cap_daily` / `spend_cap_monthly`. Carries `retryAfter`. |
| `RateLimited` | `VendoError` | `upstream_rate_limited`. Carries `retryAfter`. |
| `UpstreamError` | `VendoError` | `upstream_error`. |
| `ValidationError` | `VendoError` | `validation_failed` (e.g. bad webhook signature). |
| `IdempotencyConflict` | `VendoError` | `idempotency_conflict`. |
| `VendoOnlyFeature` | `VendoError` | `vendo_only_feature` — called a Vendo-only method in OSS mode. |
| `IdentityNotPresent` | `VendoError` | `identity_not_present` — `forRequest()` called without `X-Vendo-User-JWT`. |

Every `VendoError` instance carries `code`, `message`, and when applicable
`status`, `slug`, `connectUrl`, `retryAfter`, `suggestedFix`.
