Generated from `packages/sdk-surface/surface.yaml`. Every public method on `@vendodev/sdk` is documented here.

All methods live on a `Vendo` client instance. Every method is `async` and returns a `Promise`. The client is always async — there is no sync variant.

```typescript
import { Vendo } from "@vendodev/sdk";
const vendo = new Vendo();
```

## client

Top-level methods on the client instance itself.

### isVendoMode

Returns `true` when the SDK is running in Vendo mode (`VENDO_API_KEY` is set).

```typescript
vendo.isVendoMode(): boolean
```

**Returns:** `boolean` — `true` in Vendo mode, `false` in OSS/BYOK mode.

**Mode behavior:**
- Vendo mode: returns `true`.
- OSS mode: returns `false`. Does not throw.

```typescript
if (vendo.isVendoMode()) {
  const balance = await vendo.billing.balance();
}
```

### forUser

Returns a new client instance scoped to a user JWT. Use in background jobs or code paths where you hold a JWT directly.

```typescript
vendo.forUser(userJwt: string): Vendo
```

**Returns:** `Vendo` — a new client that injects `X-Vendo-User-JWT` on every outbound request.

**Throws:** `VendoOnlyFeature` — in OSS mode.

**Mode behavior:**
- Vendo mode: clones the client with the JWT attached.
- OSS mode: throws `VendoOnlyFeature`.

```typescript
const client = vendo.forUser(userJwt);
const conn = await client.connections.get("stripe");
```

---

### forRequest

Returns a new client scoped to the `X-Vendo-User-JWT` extracted from an incoming request's headers.

```typescript
vendo.forRequest(headers: Headers | Record<string, string>): Vendo
```

**Returns:** `Vendo` — a new client scoped to the user JWT in the headers.

**Throws:**
- `VendoOnlyFeature` — in OSS mode.
- `IdentityNotPresent` — when `X-Vendo-User-JWT` is absent from the headers.

**Mode behavior:**
- Vendo mode: reads `X-Vendo-User-JWT` (case-insensitive) and returns a scoped client.
- OSS mode: throws `VendoOnlyFeature`.

```typescript
// In a Next.js route handler:
const client = vendo.forRequest(request.headers);
const conn = await client.connections.get("notion");
```

### token

Returns the credential token for a provider slug. In Vendo mode this fetches a refreshed OAuth token; in OSS mode it reads the conventional env var.

```typescript
vendo.token(slug: string): Promise<string>
```

**Returns:** `Promise<string>` — the credential token.

**Throws:** `NotConnected` — when no token is found.

**Mode behavior:**
- Vendo mode: calls `credentials.vendo.run`, caches in-process.
- OSS mode: reads the `primary` env var from the bundled BYOK map. Override any slug with `VENDO_TOKEN_<UPPER_SLUG>`.

```typescript
const openaiKey = await vendo.token("openai");
```

---

### tokens

Returns tokens for multiple slugs in one call.

```typescript
vendo.tokens(slugs: string[]): Promise<Record<string, string | null>>
```

**Returns:** `Promise<Record<string, string | null>>` — mapping from slug to token or `null`.

**Mode behavior:**
- Vendo mode: fans out to `credentials.vendo.run`.
- OSS mode: reads env vars for each slug; unmapped slugs return `null`.

```typescript
const tokens = await vendo.tokens(["openai", "anthropic"]);
// { openai: "sk-...", anthropic: null }
```

### invalidate

Invalidates the cached token for a slug, forcing the next `token()` call to re-fetch.

```typescript
vendo.invalidate(slug: string): void
```

**Returns:** `void`

**Mode behavior:**
- Vendo mode: clears the in-process cache entry.
- OSS mode: no-op.

```typescript
vendo.invalidate("google");
```

vendo.connectUrl(slug: string, options?: { returnTo?: string; state?: string }): string

## connections

Read the user's provider connections for this deployment.

### connections.list

Returns all connections for this deployment, both connected and available.

```typescript
vendo.connections.list(): Promise<Connection[]>
```

**Returns:** `Promise<Connection[]>` — every supported connection. Entries with `status === "available"` have no active binding.

**Mode behavior:**
- Vendo mode: calls `GET /api/deployments/me/connections`.
- OSS mode: derives connections from env vars. Each entry has `profile: "byok_static"`, no `id`, no `connectedAt`.

```typescript
const all = await vendo.connections.list();
const connected = all.filter(c => c.status === "connected");
```

---

### connections.get

Returns a single connection by provider slug, or `null` if the user has not connected it.

```typescript
vendo.connections.get(slug: string): Promise<Connection | null>
```

**Returns:** `Promise<Connection | null>` — `null` means not connected (do not confuse with `NotConnected`, which is thrown by `data.execute`).

**Mode behavior:**
- Vendo mode: fetches from `/api/deployments/me/connections` and filters by slug.
- OSS mode: checks the BYOK env var for the slug.

```typescript
const conn = await vendo.connections.get("supabase");
if (!conn) {
  return { needsConnection: "supabase" };
}
const ref = conn.context?.["ref"];
```

## events

Server-side SSE event stream. Vendo mode only.

### events.subscribe

Opens a persistent SSE stream and yields events as they arrive. Auto-reconnects with exponential backoff.

```typescript
vendo.events.subscribe(): AsyncIterable<Event>
```

**Returns:** `AsyncIterable<Event>`

**Throws:** `VendoOnlyFeature` — in OSS mode.

**Mode behavior:** Vendo mode only. Falls back to polling when SSE errors three times in a row.

```typescript
for await (const event of vendo.events.subscribe()) {
  if (event.kind === "connection.connected") {
    vendo.invalidate(event.slug);
  }
}
```

## data

Execute Composio-backed actions against connected providers. Vendo mode only.

### data.execute

Posts an action to the Composio proxy and returns the result.

```typescript
vendo.data.execute(action: string, args?: Record<string, unknown>): Promise<Record<string, unknown>>
```

**Returns:** `Promise<Record<string, unknown>>` — the Composio action response payload.

**Throws:**
- `NotConnected` — when the provider required by the action is not connected.
- `VendoOnlyFeature` — in OSS mode.
- `UpstreamError` — when Composio returns an error.
- `VendoTimeout` — when the action exceeds the timeout (default 5s in sandbox).

**Mode behavior:** Vendo mode only. Posts to `POST /v1/composio-proxy/execute`.

```typescript
const result = await vendo.data.execute(
  "SUPABASE_BETA_RUN_SQL_QUERY",
  { project_id: "vqxkddreoeebocpisgrn", query: "SELECT count(*) FROM auth.users" }
);
```

## reconciler

File-based env-var reconciler. Keeps a `.env` file in sync with active connections.

### reconciler.bootstrap

Writes a `.env` file once from the current connection state.

```typescript
vendo.reconciler.bootstrap(options: { envFile: string; mapping: Record<string, unknown> }): Promise<void>
```

**Mode behavior:** Works in any mode.

```typescript
const { mapping } = await vendo.reconciler.catalogDefault();
await vendo.reconciler.bootstrap({ envFile: "/app/.env", mapping });
```

---

### reconciler.start

Starts a long-running watcher that keeps the `.env` file in sync.

```typescript
vendo.reconciler.start(options: {
  envFile: string;
  mapping: Record<string, unknown>;
  onChange?: () => void;
  reloadUrl?: string;
}): Promise<Reconciler>
```

**Returns:** `Promise<Reconciler>` — call `.stop()` to shut down the watcher.

**Mode behavior:** Works in any mode. In Vendo mode, SSE events drive immediate invalidation.

```typescript
const { mapping, restartHandlerFactory } = await vendo.reconciler.catalogDefault();
const reconciler = await vendo.reconciler.start({
  envFile: "/app/.env",
  mapping,
  onChange: restartHandlerFactory({ gateway: () => process.exit(0) }),
});
```

---

## Types

### Connection

Represents a provider connection for this deployment.

| Field | Type | Description |
|---|---|---|
| `id` | `string \| null` | UUID. `null` on `available` entries. |
| `slug` | `string` | Provider slug, e.g. `"supabase"`, `"stripe"`. |
| `displayName` | `string` | Human-readable provider name. |
| `category` | `string` | Provider category. |
| `profile` | `string` | Credential management mode. |
| `status` | `"connected" \| "available" \| "needs_reauth" \| "error"` | Connection state. |
| `context` | `Record<string, unknown> \| null` | Flattened discovery identifiers (e.g. `{ ref: "vqxkddreoeebocpisgrn" }` for Supabase). `null` for non-discovery providers. |
| `apiKey` | `string \| null` | Set only for `vendo_managed_pool` connections. |
| `baseUrl` | `string \| null` | Set only for `vendo_managed_pool` connections. |
| `metadata` | `Record<string, unknown>` | Raw provider metadata. Prefer `context` over reading from here directly. |
| `setupUrl` | `string \| null` | OAuth connect URL. Set on `available` entries. |
| `errorMessage` | `string \| null` | Human-readable error when `status` is `"error"` or `"needs_reauth"`. |
| `envBootstrap` | `object \| null` | Env-var bootstrap spec for the reconciler. |
| `exclusive` | `boolean \| undefined` | `true` when only one instance can hold this credential (e.g. Telegram). |
| `logoUrl` | `string \| undefined` | Provider logo URL. |
| `brandColor` | `string \| undefined` | Provider brand color (hex). |
| `docsUrl` | `string \| undefined` | Provider docs URL. |
| `connectedAt` | `string \| null` | ISO 8601 timestamp. |

Example JSON (TypeScript camelCase keys):

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "slug": "supabase",
  "displayName": "Supabase",
  "category": "database",
  "profile": "user_oauth",
  "status": "connected",
  "context": { "ref": "vqxkddreoeebocpisgrn" },
  "apiKey": null,
  "baseUrl": null,
  "metadata": {},
  "setupUrl": null,
  "errorMessage": null,
  "envBootstrap": null,
  "connectedAt": "2026-05-19T12:34:56Z"
}
```

**Context field by provider:**

| Provider | `context` shape |
|---|---|
| `supabase` | `{ ref: "<project-ref>" }` |
| `instagram` | `{ igUserId: "<id>" }` |
| Slack, Gmail, Stripe, etc. | `null` |

## Errors

| Class | Extends | When thrown |
|---|---|---|
| `VendoError` | `Error` | Base class for all SDK errors. |
| `AuthError` | `VendoError` | Bad or expired API key. |
| `NotConnected` | `VendoError` | Provider not connected or env var missing. Carries `slug` and `suggestedEnvVars`. |
| `NeedsReauth` | `VendoError` | Connection exists but requires re-authorization. |
| `BalanceExhausted` | `VendoError` | Credit balance is zero. |
| `SpendCapExceeded` | `VendoError` | A spend cap was hit. |
| `RateLimited` | `VendoError` | Too many requests. Carries `retryAfter`. |
| `UpstreamError` | `VendoError` | Upstream provider returned an error. |
| `ValidationError` | `VendoError` | Input failed validation (e.g. bad webhook signature). |
| `IdempotencyConflict` | `VendoError` | Duplicate idempotency key with different body. |
| `VendoOnlyFeature` | `VendoError` | Called a Vendo-only method in OSS mode. |
| `IdentityNotPresent` | `VendoError` | `forRequest()` called without `X-Vendo-User-JWT` header. |
| `VendoTimeout` | `VendoError` | Operation exceeded the timeout. |
