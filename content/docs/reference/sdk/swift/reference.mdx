Sourced from `vendo-sdk-swift` (`Sources/Vendo/`). Every public method on the
`Vendo` actor is documented here, plus the `BYOK` namespace and shared model
types.

```swift
import Vendo

let vendo = try Vendo()
```

`Vendo` is an `actor`. Sub-API properties (`connections`, `billing`, etc.) are
declared `nonisolated`, so synchronous access does not require `await`; the
actor's own async methods (`token`, `forUser`, etc.) do.

---

## Vendo (init)

```swift
public init(
    apiKey: String? = nil,
    baseURL: String? = nil,
    apiVersion: String = "2026-05-07",
    userJWT: String? = nil,
    env: [String: String] = ProcessInfo.processInfo.environment
) throws
```

- `apiKey` â€” falls back to `env["VENDO_API_KEY"]`. Throws
  `VendoError.auth(message:code:)` when both are absent / empty.
- `baseURL` â€” falls back to `env["VENDO_BASE_URL"]`, then
  `"https://vendo.run"`. Every call site prefixes `/api/â€¦`, so do **not** include
  `/api` in `baseURL`.
- `apiVersion` â€” sent as `Vendo-API-Version` header.
- `userJWT` â€” when set, every request adds `X-Vendo-User-JWT`.
- `env` â€” pass a controlled dict in tests.

```swift
let vendo = try Vendo(apiKey: "vendo_sk_...")
```

## token

```swift
public func token(_ slug: String) async throws -> String
```

Resolves a token. Resolution order:

1. `VENDO_TOKEN_<UPPER_SLUG>` env override (any mode).
2. Vendo mode (`VENDO_API_KEY` set): in-memory cache, then
   `credentials.vendo.run/<slug>`.
3. BYOK mode: primary env var from the bundled `byok.json`.
4. Throws `VendoError.notConnected(slug:message:)` if nothing matches.

```swift
let openai = try await vendo.token("openai")
```

## tokens

```swift
public func tokens(_ slugs: [String]) async throws -> [String: String?]
```

Bulk resolution. Slugs with no token map to `nil`.

```swift
let map = try await vendo.tokens(["openai", "anthropic"])
// ["openai": "sk-...", "anthropic": nil]
```

## invalidate

```swift
public func invalidate(_ slug: String)
```

Evicts the cached token for a slug. Async (because `Vendo` is an actor) but
performs no I/O.

## forUser

```swift
public func forUser(jwt: String) throws -> Vendo
```

Returns a new `Vendo` instance whose every request injects
`X-Vendo-User-JWT: <jwt>`.

## forRequest

```swift
public func forRequest(headers: [String: String]) throws -> Vendo
```

Reads `X-Vendo-User-JWT` (case-insensitive) and returns a user-scoped
`Vendo` instance.

**Throws:**
- `VendoError.vendoOnlyFeature` â€” in OSS mode.
- `VendoError.identityNotPresent` â€” when the header is absent.

## events.subscribe

```swift
try vendo.events.subscribe(maxBackoffSeconds: TimeInterval = 30) -> EventsAsyncSequence
```

`EventsAsyncSequence` is an `AsyncSequence` of `EventStreamMessage`. Iteration
auto-reconnects on transient errors with exponential backoff capped at
`maxBackoffSeconds`.

```swift
for try await event in try vendo.events.subscribe() {
    print(event.type, event.data)
}
```

## webhooks (WebhooksAPI)

```swift
public init(secret: String? = nil, maxAgeSeconds: Int = 300)

public func verify(headers: [String: String], body: String) throws -> WebhookEvent
```

`secret` and `maxAgeSeconds` are constructor parameters. `verify(...)` takes
only `(headers, body)`. When `secret:` is omitted, the verifier reads
`VENDO_WEBHOOK_SECRET` from `ProcessInfo.processInfo.environment`.

```swift
let webhooks = WebhooksAPI(secret: "whsec_...")
let event = try webhooks.verify(headers: requestHeaders, body: rawBody)
```

### AnyCodable

Type-erased JSON wrapper used by `EventStreamMessage.data`,
`WebhookEvent.data`, `Connection.metadata`, and `billing.usage`. Decodes
`null`, `Bool`, `Int`, `Double`, `String`, `[Any]`, `[String: Any]`. Integer
JSON values decode as `Int`; fractional values as `Double`.

## VendoError

```swift
public enum VendoError: Error, Sendable {
    case auth(message: String, code: String)
    case notConnected(slug: String?, message: String)
    case needsReauth(slug: String?, message: String, connectURL: String?)
    case balanceExhausted(message: String)
    case spendCapExceeded(message: String, retryAfter: Int?)
    case rateLimited(retryAfter: Int?)
    case upstream(status: Int, message: String)
    case validation(message: String)
    case idempotencyConflict(message: String)
    case vendoOnlyFeature(featureName: String)
    case identityNotPresent(message: String)
    case other(code: String, message: String, status: Int?)
}
```

Use `VendoError.from(status:headers:body:)` to convert a raw HTTP response
into the matching case. The mapping uses the `Vendo-Error-Code` header (or
the body's `error.code`):

| Code | Case |
|---|---|
| `app_unknown`, `app_revoked`, `app_expired` | `.auth(message:, code:)` |
| `binding_missing`, `connection_revoked` | `.notConnected(slug:, message:)` |
| `connection_needs_reauth` | `.needsReauth(slug:, message:, connectURL:)` |
| `balance_exhausted` | `.balanceExhausted(message:)` |
| `spend_cap_daily`, `spend_cap_monthly` | `.spendCapExceeded(message:, retryAfter:)` |
| `upstream_rate_limited` | `.rateLimited(retryAfter:)` |
| `upstream_error` | `.upstream(status:, message:)` |
| `validation_failed` | `.validation(message:)` |
| `idempotency_conflict` | `.idempotencyConflict(message:)` |
| `vendo_only_feature` | `.vendoOnlyFeature(featureName:)` |
| `identity_not_present` | `.identityNotPresent(message:)` |
| any other | `.other(code:, message:, status:)` |
