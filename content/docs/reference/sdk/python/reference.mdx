## client

Top-level methods on the client instance itself.

### is_vendo_mode

Returns `True` when the SDK is running in Vendo mode (`VENDO_API_KEY` is set).

```python
vendo.is_vendo_mode() -> bool
```

**Returns:** `bool` â€” `True` in Vendo mode, `False` in OSS/BYOK mode.

**Mode behavior:**
- Vendo mode: returns `True`.
- OSS mode: returns `False`. Does not raise.

```python
if vendo.is_vendo_mode():
    balance = vendo.billing.balance()
```

### for_user

Returns a new client instance scoped to a user JWT. Use in background jobs or any code path where you hold a JWT directly.

```python
vendo.for_user(user_jwt: str) -> Vendo
```

**Returns:** `Vendo` â€” a new client that injects `X-Vendo-User-JWT` on every outbound request.

**Raises:** `VendoOnlyFeature` â€” in OSS mode. Multi-tenant scoping requires a Vendo backend.

**Mode behavior:**
- Vendo mode: clones the client with the JWT attached.
- OSS mode: raises `VendoOnlyFeature`.

```python
client = vendo.for_user(user_jwt)
conn = client.connections.get("stripe")
```

### for_request

Returns a new client scoped to the `X-Vendo-User-JWT` extracted from an incoming request's headers.

```python
vendo.for_request(headers: dict) -> Vendo
```

**Returns:** `Vendo` â€” a new client scoped to the user JWT in the headers.

**Raises:**
- `VendoOnlyFeature` â€” in OSS mode.
- `IdentityNotPresent` â€” when `X-Vendo-User-JWT` is absent from the headers.

**Mode behavior:**
- Vendo mode: reads `X-Vendo-User-JWT` (case-insensitive) and returns a scoped client.
- OSS mode: raises `VendoOnlyFeature`.

```python
# In a FastAPI handler:
client = vendo.for_request(dict(request.headers))
conn = client.connections.get("notion")
```

### token

Returns the credential token for a provider slug. In Vendo mode this fetches a refreshed OAuth token; in OSS mode it reads the conventional env var.

```python
vendo.token(slug: str) -> str
```

**Returns:** `str` â€” the credential token.

**Raises:** `NotConnected` â€” when no token is found (Vendo: no active connection; OSS: env var not set).

**Mode behavior:**
- Vendo mode: calls `credentials.vendo.run`, caches in-process.
- OSS mode: reads the `primary` env var from the bundled BYOK map (e.g. `OPENAI_API_KEY` for `"openai"`). Override any slug with `VENDO_TOKEN_<UPPER_SLUG>`.

```python
openai_key = vendo.token("openai")
```

### tokens

Returns tokens for multiple slugs in one call. Slugs with no token return `None`.

```python
vendo.tokens(slugs: list[str]) -> dict[str, str | None]
```

**Returns:** `dict[str, str | None]` â€” mapping from slug to token or `None`.

**Mode behavior:**
- Vendo mode: fans out to `credentials.vendo.run` (or a bulk endpoint if available).
- OSS mode: reads env vars for each slug independently. Slugs with no env var map to `None`.

```python
tokens = vendo.tokens(["openai", "anthropic"])
# {"openai": "sk-...", "anthropic": None}
```

### invalidate

Invalidates the cached token for a slug, forcing the next `token()` call to re-fetch.


**Returns:** `None`

**Mode behavior:**
- Vendo mode: clears the in-process cache entry.
- OSS mode: no-op (no cache).


## connections

Read the user's provider connections for this deployment.

### connections.list

Returns all connections for this deployment, both connected and available.

```python
vendo.connections.list() -> list[Connection]
```

**Returns:** `list[Connection]` â€” every supported connection. Entries with `status = "available"` have no active binding.

**Mode behavior:**
- Vendo mode: calls `GET /api/deployments/me/connections`. Returns the full list.
- OSS mode: derives connections from env vars that are set. Each entry has `profile = "byok_static"`, no `id`, no `connected_at`.

```python
all_conns = vendo.connections.list()
connected = [c for c in all_conns if c.status == "connected"]
```

---

### connections.get

Returns a single connection by provider slug, or `None` if the user has not connected it.

```python
vendo.connections.get(slug: str) -> Connection | None
```

**Returns:** `Connection | None` â€” `None` means the provider is not connected (do not confuse with `NotConnected`, which is raised by `data.execute`).

**Mode behavior:**
- Vendo mode: fetches from `/api/deployments/me/connections` and filters by slug.
- OSS mode: checks the BYOK env var for the slug.

```python
conn = vendo.connections.get("supabase")
if conn is None:
    return {"needs_connection": "supabase"}
ref = conn.context["ref"]
```

## events

Server-side SSE event stream. Vendo mode only.

### events.subscribe

Opens a persistent SSE stream and yields events as they arrive. Auto-reconnects with exponential backoff.

```python
vendo.events.subscribe() -> AsyncIterable[Event]
```

**Returns:** `AsyncIterable[Event]`

**Raises:** `VendoOnlyFeature` â€” in OSS mode.

**Mode behavior:** Vendo mode only. Falls back to polling when the SSE endpoint returns 404 or errors three times in a row.

```python
async with vendo.events.subscribe() as stream:
    async for event in stream:
        if event.kind == "connection.connected":
            vendo.invalidate(event.slug)
```

## data

Execute Composio-backed actions against connected providers. Vendo mode only.

### data.execute

Posts an action to the Composio proxy and returns the result. The proxy validates the bearer, resolves the connected account, calls Composio, bills via the `composio.action_call` meter, and returns the response.

```python
vendo.data.execute(action: str, args: dict | None = None) -> dict
```

**Returns:** `dict` â€” the Composio action response payload.

**Raises:**
- `NotConnected` â€” when the provider required by the action is not connected (`binding_missing` from the proxy).
- `VendoOnlyFeature` â€” in OSS mode.
- `UpstreamError` â€” when Composio returns an error.
- `VendoTimeout` â€” when the action does not complete within the timeout (default 5s in sandbox; proxy default is higher).

**Mode behavior:** Vendo mode only. Posts to `POST /v1/composio-proxy/execute` via `VENDO_API_URL`.

```python
result = vendo.data.execute(
    "SUPABASE_BETA_RUN_SQL_QUERY",
    {"project_id": "vqxkddreoeebocpisgrn", "query": "SELECT count(*) FROM auth.users"}
)
```

File-based env-var reconciler. Keeps a `.env` file in sync with the deployment's active connections.

### reconciler.bootstrap

Writes a `.env` file once from the current connection state.

```python

```

**Returns:** `None`

**Mode behavior:** Works in any mode.

```python

```

---

### reconciler.start

Starts a long-running watcher that keeps the `.env` file in sync and optionally calls `on_change` or hits `reload_url` when the file changes.

```python
vendo.reconciler.start(
    *,
    env_file: str,
    mapping: dict,
    on_change: Callable | None = None,
    reload_url: str | None = None,
) -> Reconciler
```

**Returns:** `Reconciler` â€” call `.stop()` to shut down the watcher.

**Mode behavior:** Works in any mode. In Vendo mode, SSE events from `events.subscribe` drive immediate invalidation instead of waiting for the poll interval.

```python
mapping, restart_factory = vendo.reconciler.catalog_default()
reconciler = vendo.reconciler.start(
    env_file="/app/.env",
    mapping=mapping,
    on_change=restart_factory(gateway=lambda: os.kill(os.getpid(), signal.SIGHUP)),
)
```


## Types

### Connection

Represents a provider connection for this deployment.

| Field | Type | Description |
|---|---|---|
| `id` | `str \| None` | UUID. `None` on `available` entries. |
| `slug` | `str` | Provider slug, e.g. `"supabase"`, `"stripe"`. |
| `display_name` | `str` | Human-readable provider name. |
| `category` | `str` | Provider category, e.g. `"database"`, `"crm"`. |
| `profile` | `str` | How the credential is managed. One of: `"vendo_managed_pool"`, `"byok_static"`, `"user_oauth"`, `"oauth_app_install"`, `"webhook_inbound"`. |
| `status` | `str` | `"connected"`, `"available"`, `"needs_reauth"`, or `"error"`. |
| `context` | `dict \| None` | Flattened discovery identifiers for this provider (e.g. `{"ref": "vqxkddreoeebocpisgrn"}` for Supabase). `None` for non-discovery providers (Slack, Gmail, etc.). |
| `api_key` | `str \| None` | Set only for `vendo_managed_pool` connections. Pass directly to the provider client. |
| `base_url` | `str \| None` | Set only for `vendo_managed_pool` connections. The Vendo proxy URL to route through. |
| `metadata` | `dict` | Raw provider metadata. The `context` field is promoted from here â€” prefer `context` over `metadata["context"]`. |
| `setup_url` | `str \| None` | OAuth connect URL. Set on `available` entries, `None` when already connected. |
| `error_message` | `str \| None` | Human-readable error. Set when `status` is `"error"` or `"needs_reauth"`. |
| `env_bootstrap` | `dict \| None` | Env-var bootstrap spec. Tells the reconciler which vars to write and when to restart. |
| `exclusive` | `bool \| None` | `True` when only one running instance can use this credential at a time (e.g. Telegram). |
| `logo_url` | `str \| None` | Provider logo URL. |
| `brand_color` | `str \| None` | Provider brand color (hex). |
| `docs_url` | `str \| None` | Provider documentation URL. |
| `connected_at` | `str \| None` | ISO 8601 timestamp of when the connection was established. |

Example JSON:

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "slug": "supabase",
  "display_name": "Supabase",
  "category": "database",
  "profile": "user_oauth",
  "status": "connected",
  "context": { "ref": "vqxkddreoeebocpisgrn" },
  "api_key": null,
  "base_url": null,
  "metadata": {},
  "setup_url": null,
  "error_message": null,
  "env_bootstrap": null,
  "exclusive": null,
  "logo_url": "https://vendo.run/logos/supabase.svg",
  "brand_color": "#3ECF8E",
  "docs_url": "https://supabase.com/docs",
  "connected_at": "2026-05-19T12:34:56Z"
}
```

**Context field by provider:**

| Provider | `context` shape |
|---|---|
| `supabase` | `{ "ref": "<project-ref>" }` |
| `instagram` | `{ "ig_user_id": "<id>" }` |
| Slack, Gmail, Stripe, etc. | `null` â€” no discovery context |

## Errors

| Class | Parent | When raised |
|---|---|---|
| `VendoError` | `Exception` | Base class for all SDK errors. |
| `AuthError` | `VendoError` | Bad or expired API key. |
| `NotConnected` | `VendoError` | Required provider is not connected or env var is missing. Carries `slug` and `suggested_env_vars`. |
| `NeedsReauth` | `VendoError` | Connection exists but requires re-authorization (OAuth token expired, revoked). |
| `BalanceExhausted` | `VendoError` | Credit balance is zero. |
| `SpendCapExceeded` | `VendoError` | A spend cap was hit. |
| `RateLimited` | `VendoError` | Too many requests. Carries `retry_after`. |
| `UpstreamError` | `VendoError` | The upstream provider (Composio, the provider API) returned an error. |
| `ValidationError` | `VendoError` | Input failed validation (e.g. bad webhook signature). |
| `IdempotencyConflict` | `VendoError` | Duplicate idempotency key with a different request body. |
| `VendoOnlyFeature` | `VendoError` | Called a Vendo-only method in OSS mode. Carries a hint message. |
| `IdentityNotPresent` | `VendoError` | `for_request()` called but `X-Vendo-User-JWT` header is absent. |
| `VendoTimeout` | `VendoError` | Operation exceeded the timeout (default 5s in sandbox). |
