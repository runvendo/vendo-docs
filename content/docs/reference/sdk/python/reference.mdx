Sourced from the `vendo` package (PyPI: `vendo-sdk`). Every public method on the
class-based client is documented here, plus the module-level helpers.

All client methods live on a `Vendo` (sync) or `AsyncVendo` (async) instance.
The examples below use the sync client; swap `Vendo` for `AsyncVendo` and
`await` the calls for async usage.

```python
import vendo
from vendo import Vendo
```

---

## client

Top-level methods on the client instance itself.

### is_vendo_mode

Returns `True` when the SDK is running in Vendo mode (`VENDO_API_KEY` is set).

```python
vendo.is_vendo_mode() -> bool
```

**Returns:** `bool` — `True` in Vendo mode, `False` in OSS/BYOK mode.

**Mode behavior:**
- Vendo mode: returns `True`.
- OSS mode: returns `False`. Does not raise.

```python
if vendo.is_vendo_mode():
    balance = vendo.billing.balance()
```

---

### for_user

Returns a new client instance scoped to a user JWT. Use in background jobs or any code path where you hold a JWT directly.

```python
Vendo.for_user(user_jwt: str) -> Vendo
```

**Returns:** `Vendo` — a new client that injects `X-Vendo-User-JWT` on every outbound request.

**Raises:** `VendoOnlyFeature` — in OSS mode. Multi-tenant scoping requires a Vendo backend.

```python
client = Vendo()
scoped = client.for_user(user_jwt)
conn = scoped.connections.get("stripe")
```

---

### for_request

Returns a new client scoped to the `X-Vendo-User-JWT` extracted from an incoming request's headers.

```python
Vendo.for_request(headers) -> Vendo
```

`headers` is any Mapping-like object — Starlette / FastAPI `Headers`, `dict`,
`werkzeug.MultiDict`, etc. Lookup is case-insensitive.

**Returns:** `Vendo` — a new client scoped to the user JWT in the headers.

**Raises:**
- `VendoOnlyFeature` — in OSS mode.
- `IdentityNotPresent` — when `X-Vendo-User-JWT` is absent from the headers.

```python
# In a FastAPI handler:
client = Vendo().for_request(request.headers)
conn = client.connections.get("notion")
```

### token

Returns the credential token for a provider slug. In Vendo mode this fetches a refreshed OAuth token; in OSS mode it reads the conventional env var.

```python
Vendo.token(slug: str) -> str
```

**Returns:** `str` — the credential token.

**Raises:** `NotConnected` — when no token is found (Vendo: no active connection; OSS: env var not set).

**Mode behavior:**
- Vendo mode: calls `credentials.vendo.run`, caches in-process.
- OSS mode: reads the `primary` env var from the bundled BYOK map (e.g. `OPENAI_API_KEY` for `"openai"`). Override any slug with `VENDO_TOKEN_<UPPER_SLUG>`.

```python
openai_key = vendo.token("openai")
```

---

### tokens

Returns tokens for multiple slugs in one call. Slugs with no token return `None`.

```python
Vendo.tokens(slugs: list[str]) -> dict[str, str | None]
```

**Returns:** `dict[str, str | None]` — mapping from slug to token or `None`.

```python
tokens = vendo.tokens(["openai", "anthropic"])
# {"openai": "sk-...", "anthropic": None}
```

---

### invalidate

Invalidates the cached token for a slug, forcing the next `token()` call to re-fetch.

```python
Vendo.invalidate(slug: str) -> None
```

```python
vendo.invalidate("google")  # force refresh after user reconnects
```

---

### connect_url

Returns the OAuth connect URL for a provider slug. Use this to send users to the Vendo connect portal.

```python
Vendo.connect_url(slug: str, *, return_to: str | None = None, state: str | None = None) -> str
```

`return_to` and `state` are keyword-only.

**Raises:** `VendoOnlyFeature` — in OSS mode.

```python
url = vendo.connect_url("stripe", return_to="https://myapp.vendo.run/dashboard")
```

## connections

Read the user's provider connections for this deployment.

### connections.list

Returns all connections for this deployment, both connected and available.

```python
Vendo.connections.list() -> list[Connection]
```

```python
all_conns = vendo.connections.list()
connected = [c for c in all_conns if c.status == "connected"]
```

---

### connections.get

Returns a single connection by provider slug, or `None` if the slug is absent from the catalog for this deployment.

```python
Vendo.connections.get(slug: str) -> Connection | None
```

`None` means the slug is not in this deployment's catalog at all. An
"available" connection (catalog-listed but unbound) is still returned with
`status == "available"` — check `c.status` to distinguish.

```python
conn = vendo.connections.get("supabase")
if conn is None:
    return {"error": "supabase not in catalog"}
if conn.status != "connected":
    return {"needs_connection": "supabase"}
ref = conn.context["ref"]
```

## events

Server-side SSE event stream. Vendo mode only.

### events.subscribe

Opens a persistent SSE stream and yields events as they arrive. Auto-reconnects with exponential backoff (capped at 30 s by default).

```python
Vendo.events.subscribe(*, max_backoff_sec: float = 30.0) -> Iterator[EventStreamMessage]
```

**Returns:** synchronous `Iterator[EventStreamMessage]` — iterate with a plain
`for` loop. The async client's `aio` namespace does not implement `subscribe`.

**Raises:** `VendoOnlyFeature` — in OSS mode (raised at call time, not on first iteration).

```python
for event in vendo.events.subscribe():
    if event.type == "connection.connected":
        vendo.invalidate(event.data.get("slug"))
```

## webhooks

Verify inbound webhook signatures. Works in any mode.

### webhooks.verify

Verifies the HMAC signature on an inbound webhook and returns the parsed event.

```python
from vendo import WebhooksAPI

hooks = WebhooksAPI(secret="whsec_...", max_age_sec=300)
hooks.verify(headers, body: str) -> WebhookEvent
```

`secret` and `max_age_sec` are **constructor** parameters. `verify()` takes only
`(headers, body)`. `headers` is any case-insensitive Mapping; `body` is the
raw request body as a `str`.

When `WebhooksAPI()` is constructed without `secret=`, the verifier reads
`VENDO_WEBHOOK_SECRET` from the environment.

**Raises:** `ValidationError` — signature missing, timestamp out of window, body not JSON, or HMAC mismatch.

```python
event = hooks.verify(dict(request.headers), await request.body())
if event.type == "connection.connected":
    ...
```

## sso (module)

`vendo.sso` is a **module-level** helper, not an attribute on the `Vendo`
class. Use it from any request handler running behind the Vendo proxy.

### sso.is_enabled

```python
vendo.sso.is_enabled() -> bool
```

Returns `True` when `VENDO_AUTH=1`.

### sso.identity_from_headers

```python
vendo.sso.identity_from_headers(headers) -> VendoIdentity | None
```

Parses trusted `X-Vendo-*` headers. Returns `None` if any required header is
missing.

### sso.identity_response

```python
vendo.sso.identity_response(headers) -> dict
```

Builds the canonical identity JSON body. Returns `{}` when headers are absent.

### sso.logout_url / sso.dashboard_url

```python
vendo.sso.logout_url() -> str          # "/__vendo/auth/logout"
vendo.sso.dashboard_url() -> str       # "https://vendo.run/dashboard"
```

### sso.IDENTITY_ENDPOINT_PATH

```python
vendo.sso.IDENTITY_ENDPOINT_PATH       # "/api/vendo/identity"
```

Recommended mount path for the identity endpoint.

## data

Execute Composio-backed actions against connected providers. Vendo mode only.

### data.execute

Posts an action to the Composio proxy and returns the result. The proxy validates the bearer, resolves the connected account, calls Composio, bills via the `composio.action_call` meter, and returns the response.

```python
Vendo.data.execute(action: str, args: dict | None = None) -> dict
```

**Raises:**
- `NotConnected` — when the provider required by the action is not connected (`binding_missing` from the proxy).
- `VendoOnlyFeature` — in OSS mode.
- `UpstreamError` — when Composio returns an error (`composio_error`).
- `AuthError` — when `VENDO_API_KEY` is missing or invalid.

Posts to `https://composio-proxy.vendo.run/v1/execute` (override via
`VENDO_DATA_PROXY_URL`).

```python
result = vendo.data.execute(
    "SUPABASE_BETA_RUN_SQL_QUERY",
    {"project_id": "vqxkddreoeebocpisgrn", "query": "SELECT count(*) FROM auth.users"}
)
```

## reconciler (module)

`vendo.reconciler` is a **module-level** helper. File-based env-var reconciler.
Keeps a `.env` file in sync with the deployment's active connections.

### reconciler.bootstrap

Writes a `.env` file once from the current connection state.

```python
vendo.reconciler.bootstrap(
    *,
    env_file: str,
    mapping: Callable[[], Mapping[str, str]],
    on_change: "restart" | "reload-hook" | Callable | None = None,
    reload_url: str | None = None,
) -> None
```

`mapping` is a **callable** returning a `{VAR_NAME: value}` dict — not the dict
itself. All keyword-only.

```python
mapping, _ = vendo.reconciler.catalog_default()
vendo.reconciler.bootstrap(env_file="/app/.env", mapping=mapping)
```

---

### reconciler.start

Starts a long-running watcher that keeps the `.env` file in sync and optionally calls `on_change` or hits `reload_url` when the file changes.

```python
vendo.reconciler.start(
    *,
    env_file: str,
    mapping: Callable[[], Mapping[str, str]],
    on_change: "restart" | "reload-hook" | Callable = "restart",
    reload_url: str | None = None,
) -> Reconciler
```

`on_change` defaults to the string sentinel `"restart"`. Other values:
`"reload-hook"` (POST to `reload_url`) or any `Callable[[diff], None]`.

```python
mapping, restart_factory = vendo.reconciler.catalog_default()
reconciler = vendo.reconciler.start(
    env_file="/app/.env",
    mapping=mapping,
    on_change=restart_factory(gateway=lambda: os.kill(os.getpid(), signal.SIGHUP)),
)
reconciler.stop()
```

---

### reconciler.catalog_default

Returns a default mapping and restart handler factory derived from the deployment's `vendo.yaml`.

```python
vendo.reconciler.catalog_default(client=None) -> tuple[Callable, Callable]
```

## Types

### Connection

Represents a provider connection for this deployment.

| Field | Type | Description |
|---|---|---|
| `id` | `str \| None` | UUID. Optional in Python because older Vendo backends (pre-v0.9.0a5) omit it from the wire payload; `None` on `available` entries. The TypeScript SDK marks this required because its minimum supported backend already emits it. |
| `external_id` | `str \| None` | Provider-side identifier (the provider's user ID, account ID, etc.). Same backend-version caveat as `id`. |
| `slug` | `str` | Provider slug, e.g. `"supabase"`, `"stripe"`. |
| `display_name` | `str` | Human-readable provider name. |
| `category` | `str` | Provider category, e.g. `"database"`, `"crm"`. |
| `profile` | `str` | How the credential is managed. One of: `"vendo_managed_pool"`, `"byok_static"`, `"user_oauth"`, `"oauth_app_install"`, `"webhook_inbound"`. |
| `status` | `str` | `"connected"`, `"available"`, `"pending_setup"`, `"needs_reauth"`, `"error"`, or `"revoked"`. |
| `metadata` | `dict` | Raw provider metadata. The `context` field is promoted from here — prefer `context`. |
| `context` | `dict \| None` | Flattened discovery identifiers (e.g. `{"ref": "vqxkddreoeebocpisgrn"}` for Supabase). `None` for non-discovery providers. |
| `credential` | `dict \| None` | Per-provider credential payload (used by `vendo_managed_pool` connections). |
| `setup_url` | `str \| None` | OAuth connect URL. Set on `available` entries, `None` when already connected. |
| `error_message` | `str \| None` | Human-readable error when `status` is `"error"` or `"needs_reauth"`. |
| `connected_at` | `str \| None` | ISO 8601 timestamp of when the connection was established. |
| `logo_url` | `str \| None` | Provider logo URL. |
| `brand_color` | `str \| None` | Provider brand color (hex). |
| `docs_url` | `str \| None` | Provider documentation URL. |
| `env_bootstrap` | `dict \| None` | Env-var bootstrap spec. Tells the reconciler which vars to write and when to restart. |

**Context field by provider:**

| Provider | `context` shape |
|---|---|
| `supabase` | `{ "ref": "<project-ref>" }` |
| `instagram` | `{ "ig_user_id": "<id>" }` |
| Slack, Gmail, Stripe, etc. | `null` — no discovery context |

## Errors

| Class | Parent | When raised |
|---|---|---|
| `VendoError` | `Exception` | Base class for all SDK errors. |
| `AuthError` | `VendoError` | `app_unknown` / `app_revoked` / `app_expired`. |
| `NotConnected` | `VendoError` | `binding_missing` / `connection_revoked`. Carries `slug` and `suggested_fix`. |
| `NeedsReauth` | `VendoError` | `connection_needs_reauth` — connection exists but requires re-authorization. Carries `connect_url`. |
| `BalanceExhausted` | `VendoError` | `balance_exhausted` — credit balance is zero. |
| `SpendCapExceeded` | `VendoError` | `spend_cap_daily` / `spend_cap_monthly`. Carries `retry_after`. |
| `RateLimited` | `VendoError` | `upstream_rate_limited`. Carries `retry_after`. |
| `UpstreamError` | `VendoError` | `upstream_error` — provider returned an error. |
| `ValidationError` | `VendoError` | `validation_failed` (e.g. bad webhook signature). |
| `IdempotencyConflict` | `VendoError` | `idempotency_conflict` — duplicate key with different body. |
| `VendoOnlyFeature` | `VendoError` | `vendo_only_feature` — called a Vendo-only method in OSS mode. |
| `IdentityNotPresent` | `VendoError` | `identity_not_present` — `for_request()` called but header absent. |

Every `VendoError` instance carries `code`, `message`, `status` (HTTP), and
when applicable `slug`, `connect_url`, `retry_after`, `suggested_fix`.
