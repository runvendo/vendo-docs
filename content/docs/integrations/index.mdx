## How a provider gets here

Each enabled row in `integrations` ships with one or more **auth profiles** — `vendo_managed_pool` (Vendo holds the key), `byok_static` (you upload yours), `oauth_app_install` / `oauth2` (per-user OAuth), or `composio_managed` (brokered through Composio). The per-provider page lists which profiles are wired up and which one is the default.

## Adding a new integration

Integrations come from the monorepo's `packages/integrations/<slug>/` modules. To propose a new one, open an issue on [`runvendo/vendo`](https://github.com/runvendo/vendo) with: provider name, what tools would consume it, and which auth profiles the provider supports.
