## Reading the credential at runtime

Instagram is composio-managed, so Vendo never holds a long-lived copy of the upstream credential — Composio does. That means `process.env.INSTAGRAM_ACCESS_TOKEN` is **not** populated in your running container; the env-var row above describes the SDK's credential-projection shape, not a Railway env var.

To read the credential from your tool, call `vendo.token("instagram")` (the SDK fetches a live access token from Composio at call time) or use the canonical [`vendo.data.execute`](/docs/guides/recipes/call-composio-action) path, which handles metering, connected-account resolution, and error normalization for you.

## Through Composio

Calls to Instagram are brokered through Vendo's Composio bridge. Your tool issues a `vendo.data.execute(ACTION, args)` call; Vendo resolves the tenant's connected Instagram account, forwards to Composio, meters one `composio.action_call` unit, and returns the result. Your code never sees the upstream credential.

See [Call a Composio Action](/docs/guides/recipes/call-composio-action) for the full pattern, including `NotConnected` handling.
